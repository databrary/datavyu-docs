===========================
create_mutually_exclusive()
===========================

.. default-domain:: rb

.. method:: create_mutually_exclusive(name, var1name, var2name)

   Mixes two column together, creating a new column.
   :meth:`create_musually_exclusive` mixes the two source columns cells
   together so that the new column includes all of the arguments from
   both source columns, as well as a new cell for each overlap and
   mixture of the two columns' cells.
   
   .. table-start
   
   .. list-table::
      :widths: 20, 40, 50
      :header-rows: 1
      
      * - Parameter
        - Type
        - Description
      * - ``name``
        - String
        - The name of the new column you wish to create
      * - ``var1name``
        - String
        - The name of the first source column that will be :term:`mutexed`
      * - ``var2name``
        - String
        - The name of the second source column that will be :term:`mutexed`
   
   .. table-end
   
   .. admonition:: Returns
   
      The new Ruby representation of the column. You must write the
      changes back to the database if you want to save them.
   
   .. admonition:: Example
   
      The following example creates a variable called "test" from two
      existing variables, "var1" and "var2", and then writes the 
      changes back to the database using :meth:`setVariable`.
   
      .. code-block:: ruby
      
         require 'Datavyu_API.rb'
         begin
            test = create_mutually_exclusive("test", "var1", "var2")
            setVariable("test", test)
         end
